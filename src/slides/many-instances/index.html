Multiple Services, One Machine

Developers have to run many services
  • Databases: PostgreSQL, MariaDB, MySQL
  • Caching Services: Redis, Memcached
  • Web Servers: Nginx, Apache, Express, Koa
One Problem: Conflicts
  • Port mappings
  • Folder configs
  • Instance / service / executable naming
  • Time wasted fixing all of this
  • Repeat this proces for all dev machines

Docker to the Rescue (again)

  • Each instance gets to run in its own VM
  • Use defaults across all instances
  • Repeatable pattern across all dev machines
